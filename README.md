# Дипломные работы — Blazor WebAssembly

[![Deploy to GitHub Pages](https://github.com/IT-GSTU/diplomas/actions/workflows/deploy.yml/badge.svg)](https://github.com/IT-GSTU/diplomas/actions/workflows/deploy.yml)

Данное веб-приложение предназначено для просмотра, добавления, редактирования и
удаления списка дипломных работ, первоначально сформированного на основании
файла `Дипломные работы ИТ-4_2026.xlsx`. Просматривать список может любой
посетитель, тогда как операции изменения данных доступны исключительно
аутентифицированному администратору. Приложение реализовано на платформе
ASP.NET Blazor WebAssembly (.NET 10) и не использует систему управления базами
данных. Развёртывание осуществляется в виде статического сайта на платформе
GitHub Pages.

Адрес действующего сайта: https://it-gstu.github.io/diplomas/

## Принцип работы

Поскольку GitHub Pages предоставляет только статические файлы, всё
функционирование приложения происходит на стороне браузера и разделяется на два
независимых направления.

Чтение данных доступно всем посетителям без какой-либо авторизации: список
дипломных работ загружается из файла `data/diplomas.json`, расположенного в
репозитории, по прямой (raw) ссылке.

Запись данных доступна только администратору. Изменения сохраняются
непосредственно в файл `data/diplomas.json` посредством GitHub Contents API.
Для входа администратор использует персональный токен доступа к платформе
GitHub. Проверка прав на запись осуществляется самой платформой GitHub: без
токена, обладающего правом записи в соответствующий репозиторий, сохранение
изменений невозможно. Таким образом обеспечивается подлинная защита данных, а
не простое сокрытие элементов управления в пользовательском интерфейсе.

```
Браузер (Blazor WASM)
   ├── чтение  → raw.githubusercontent.com/<owner>/<repo>/<branch>/data/diplomas.json
   └── запись  → api.github.com (Contents API, с использованием токена администратора)
```

## Структура репозитория

```
.
├─ .github/workflows/deploy.yml         автоматическая публикация на платформе GitHub Pages
├─ data/diplomas.json                   данные приложения (изменяются администратором через интерфейс)
├─ tools/Convert-Xlsx.ps1               сценарий преобразования файла Excel в формат JSON
└─ DiplomasViewer/                      проект Blazor WebAssembly
   ├─ wwwroot/appsettings.json          параметры репозитория, используемого в качестве хранилища данных
   ├─ wwwroot/sample-data/diplomas.json встроенная резервная копия данных, применяемая при недоступности основного источника
   ├─ Models/                           классы Diploma и GitHubOptions
   ├─ Services/                         классы GitHubClient, DiplomaService и AdminState
   ├─ Pages/                            страницы Home (список работ) и Login (вход администратора)
   └─ Components/DiplomaEditDialog.razor форма добавления и редактирования записи
```

## Настройка для использования с другим репозиторием

Для размещения приложения в собственном репозитории необходимо отредактировать
файл `DiplomasViewer/wwwroot/appsettings.json` следующим образом:

```json
{
  "GitHub": {
    "Owner": "IT-GSTU",
    "Repo": "diplomas",
    "Branch": "main",
    "DataPath": "data/diplomas.json"
  }
}
```

Значение атрибута `base href`, требуемое для корректной работы на платформе
GitHub Pages, определяется автоматически в процессе публикации на основании
имени репозитория (в виде `/<repo>/`) и не нуждается в ручном изменении.

## Запуск в локальной среде

```bash
cd DiplomasViewer
dotnet run
```

После выполнения команды следует открыть адрес, указанный в консоли. При
отсутствии токена администратора доступен только просмотр данных.

## Публикация на платформе GitHub Pages

1. Создайте новый репозиторий (например, `IT-GSTU/diplomas`) и поместите в него
   содержимое данной папки, отправив изменения в ветку `main`.
2. В настройках репозитория выберите раздел **Settings → Pages → Build and
   deployment** и установите значение источника **GitHub Actions**.
3. Файл `deploy.yml` автоматически выполнит сборку и публикацию сайта при
   каждой отправке изменений в ветку `main`.
4. После публикации сайт станет доступен по адресу
   `https://<owner>.github.io/<repo>/`.

## Вход администратора и использование токена доступа

1. Перейдите в раздел **Settings → Developer settings → Personal access
   tokens** на платформе GitHub.
2. Рекомендуется создать токен с точно определёнными правами доступа — такой
   тип токена в GitHub называется fine-grained. Ему следует предоставить доступ
   только к необходимому репозиторию и право **Contents: Read and write**.
   Также допускается использование токена классического типа (classic) с
   областью действия `public_repo`.
3. В приложении откройте раздел «Вход администратора», введите токен и нажмите
   кнопку «Войти». После успешной авторизации станут доступны элементы
   управления для добавления, редактирования и удаления записей.

Токен сохраняется исключительно в пределах текущей вкладки браузера (в
хранилище `sessionStorage`), не передаётся в какие-либо внешние системы и не
включается в исходный код приложения. Передача токена третьим лицам
недопустима.

## Обновление исходных данных из файла Excel

Поместите файл с расширением `.xlsx` в корневую директорию репозитория и
выполните следующую команду:

```powershell
powershell -File ./tools/Convert-Xlsx.ps1
```

В результате выполнения сценария будут пересозданы файл `data/diplomas.json` и
встроенная резервная копия данных. Файл должен содержать следующие столбцы:
«Группа», «Студент», «Тема», «Руководитель», «Краткое описание», «Ссылка на
исходный код», «Ссылка на установочный файл», «Ссылка на демонстрацию», «Год».

## Замечания относительно безопасности

Действительная защита операций записи обеспечивается на стороне платформы
GitHub; проверка, выполняемая на стороне клиента, лишь определяет, какие
элементы интерфейса отображаются пользователю.

Данные, размещённые в публичном репозитории, доступны для чтения всем
пользователям, что соответствует характеру приложения, поскольку список
дипломных работ является открытым.

Прямые (raw) ссылки на файлы кэшируются сетью доставки содержимого на срок
приблизительно до пяти минут. Вследствие этого администратор видит внесённые им
изменения немедленно, поскольку данные считываются через API, тогда как прочие
пользователи увидят обновления после истечения срока действия кэша.
